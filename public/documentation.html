This API allows the browsing of movies, including viewing information, favoriting, deleting favorites.s

Introduction

Welcome to the ArtCine API! This API provides endpoints for CRUD operations (Create, Read, Update, Delete) for movie data. Below you'll find a detailed guide for each endpoint.

Base URL

All endpoints stem from the base URL: ***

Description Endpoint HTTP Method Parameters Request Example Response Example
List of all movies /movies GET None GET /movies JSON response with a list of movies
Single movie details /movies/:title GET title GET /movies/Inception JSON response with details about the movie, including title, description, genre, director, actors, image URL, and whether or not the movie is favorited by the user
Details about a genre /movies/genre/:name GET name GET /genre/Thriller JSON response with details about the genre
Details about a director /movies/director/:name GET name GET /director/Christopher%20Nolan JSON response with details about the director
User registration /users POST username, password POST /users {"username": "JohnDoe", "password": "pass123"} JSON response confirming user registration
Update user details /users/:username PUT username PUT /users/JohnDoe {"newUsername": "JaneDoe"} JSON response confirming username update
Add movie to favorites /users/:username/movies/:movieId POST username, movieId POST /users/JohnDoe/movies/60a7b2e JSON response confirming the movie has been added to favorites
Remove movie from favorites /users/:username/favorites/:movieId DELETE username, movieId DELETE /favorites/JohnDoe/42 Text confirming movie has been removed from favorites
Delete a user /users/:username DELETE username DELETE /users/JohnDoe {"message": "User JohnDoe was deleted", "status": 200}

Additional Notes

Ensure that your requests include proper authentication headers if needed. Any queries must be URL-encoded.

Authentication

Currently, no authentication is required to access the API endpoints. However, this will change in a future version. All requests to secure endpoints must be accompanied by an authentication header.
